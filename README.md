### Estructura de directorios

2. src
    - core
        - data
            - configuration
            - repository
        - interfaces
            - repository
            - service
        - models
        - services
        - viewModels
    - ui

# assets
    vdeFiles
        base de dato de ejemplo para, asi poder hacer el demo. 

# src
    contiene el código de la aplicación 

# core
    contiene todas la programación de la aplicación
    data, interface, models, services y viewModels

# ui (user interface)
    contiene todos los archivos de las interfase del usuario (*.xaml)
